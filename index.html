How does javascript work?

  • Javascript is a single threaded programming language. So it can execute or handle single task at a time.
  • Whenever we want to run a JavaScript file ,first it is received by parser.
  • JavaScript is an interpretive scripting language. So the parser checks the script line by line.
  • After syntactical error checking the code is converted into machine code.
  • At the end the code is sent to the system or operator for execution.

What does Javascript event loop do?

  • We know that Javascript is a single threaded programming language which can execute single task at a time so there should be a mechanism to make every process isolated so that the heavy processing code block does not entirely stop the code execution.
  • Event loop is the environment to manage the concurrent task without blocking others.
  • The event loop continuously checks the callback queue and call stack(LIFO process) to see if there's any function that needs to run.
  • If the call stack is not empty, the event loop waits until it is empty and places the next function from the callback queue to the call stack.
  • If the callback queue is empty, no execution will happen.

What is the difference between sessionStorage and localStorage?

  • LocalStorage data does not expire, on the other hand sessionStorage data is cleared when the page session ends.
  • LocalStorage must be deleted via javascript or manually but sessionStorage data is cleared when the browser is closed.
  • All the method(getItem,setItem) are same except we have to explicitly call the localStorage and sessionStorage.
  • The storage capacity of local storage is 10MB but the storage capacity of sessionstorage not more than 5MB.
  • Both are accessible from the client-side only which means web servers can't access them directly.
